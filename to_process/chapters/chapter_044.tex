\section{The RSVP Update Equation in Parallel Form}

The continuum RSVP evolution equations on $\mathcal{M}$ are:
\[
\partial_t \Phi = \Delta_g \Phi - \nabla\!\cdot \mathbf{v} + F_\Phi(\Phi,\mathbf{v},S),
\]
\[
\partial_t \mathbf{v} = -\nabla \Phi - \gamma \mathbf{v} + F_v(\Phi,\mathbf{v},S),
\]
\[
\partial_t S = \eta \Delta_g S + F_S(\Phi,\mathbf{v},S).
\]

GPU implementation requires:

\begin{enumerate}
\item finite-difference Laplacians $\Delta_h$ compatible with $g$,
\item parallel gradient and divergence operators $\nabla_h$ and $\nabla_h\!\cdot$,
\item synchronous update semantics ensuring modal safety.
\end{enumerate}

The GPU discretization is defined by:
\[
\Phi_i^{t+1} = \Phi_i^{t} + \Delta t\left[\Delta_h\Phi_i^{t} - (\nabla_h\!\cdot\mathbf{v})_i^{t} + F_\Phi^h(i)\right],
\]
\[
\mathbf{v}_i^{t+1} = \mathbf{v}_i^{t} + \Delta t\left[-(\nabla_h\Phi)_i^{t} - \gamma\mathbf{v}_i^{t} + F_v^h(i)\right],
\]
\[
S_i^{t+1} = S_i^{t} + \Delta t\left[\eta\,\Delta_h S_i^{t} + F_S^h(i)\right].
\]

Each $F_\bullet^h$ is a curvature-corrected parallel stencil defined so as to preserve modal correctness (Section~\ref{ch:44-modal}).

\section{CUDA-Level Semantics and Kernel Structure}

The GPU kernel uses:
\begin{itemize}

void rsvp_step_kernel(
    FieldConfig_d in, FieldConfig_d out,
    Geometry_d geom, float dt
){
    int i = threadIdx.x + blockIdx.x * blockDim.x;

    LocalFrame f = geom.frame[i];
    float Phi = in.Phi[i];
    Vec3 v = in.v[i];
    float S = in.S[i];

    float lapPhi = laplacian(Phi, geom, i);
    float divv   = divergence(v, geom, i);
    float lapS   = laplacian(S, geom, i);

    Vec3  Fv   = vector_matching_term(i,in,geom);
    float Fs   = entropy_alignment_term(i,in,geom);

    out.Phi[i] = Phi + dt*(lapPhi - divv + Fphi);
    out.v[i]   = v   + dt*(-gradient(Phi,geom,i) - gamma*v + Fv);
    out.S[i]   = S   + dt*(eta*lapS + Fs);
}
\end{verbatim}

This kernel is implemented in double precision with deterministic memory ordering.
