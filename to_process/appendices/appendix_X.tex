\section{Lean Foundations and Type Universes}

All formalisation uses Lean~4 with \texttt{mathlib}.  
Type universes are:
\[
\mathsf{FieldConfig} : \mathsf{Type}_{u_1},\qquad
\mathsf{PolyxGraph} : \mathsf{Type}_{u_2},\qquad
\mathsf{EmbedConfig} : \mathsf{Type}_{u_3}.
\]
Dependent types encode the harmonic constraint:
\[
\mathsf{RSVPState}
  :=
  \{ (\varphi,\mathbf{v},S) : \mathsf{FieldConfig} \mid
     \mathsf{Harmonic}(\varphi,\mathbf{v},S)\}.
\]

\section{Modal Logic Layer}

\subsection{Harmonic Necessity}

```lean
def Box_RSVP (P : RSVPField → Prop) : Prop :=
  ∀ f, HarmonicEvolution f → P f
```

\subsection{Quine-Stable Necessity}

```lean
def Box_Polyx (P : PolyxGraph → Prop) : Prop :=
  ∀ G G', EHRStep G G' → P G → P G'
```

\section{Guidelines for Extensions}

\begin{enumerate}
\item Modal operators must remain preserved.
\item Curvature and entropy invariants should be encoded at the type level.
\item Reconciliation is the default composition operator.
\item Duality must be extended with any new structures.
\item Proofs should be maintained in fully normalised form.
\end{enumerate}
