The core claim (de-satirized)
Safetyist alignment metaphors are inherited from Unix-era control metaphors, not invented by modern AI safety.
Specifically:
Permission
Isolation
Capability restriction
Auditability
Sandboxing
Least privilege
Fail-closed behavior
These are alignment metaphors before they were moralized.
Unix didn't ask:
"Is this process aligned with human values?"
It asked:
"What is this process allowed to do, and what happens if it misbehaves?"
That's already alignment — just without the rhetoric.
Where the "safetyism" shows up concretely
User / root ≈ human / AI
Root is not better — it's more dangerous
Root access is tightly controlled, logged, and discouraged
Most of Unix security is about preventing escalation
That maps directly onto:
"Superintelligence risk"
"Capability overhang"
"Recursive self-improvement panic"
Unix solved this by not trusting power, not by teaching values.
Exit codes as moral judgments
Unix processes don't argue ethics.
They return exit status.
0 → acceptable
non-zero → something went wrong
specific codes → constrained interpretations
This is:
Reward functions
Loss metrics
Evaluation harnesses
Your key insight:
The moment you reduce behavior to a metric, the metric becomes the game.
Unix already knows this — which is why:
Exit codes are minimal
Logs are separated from control
Success ≠ correctness
Modern alignment keeps forgetting this lesson.
Pipelines = instrumental convergence
Unix pipelines:
cat data | grep pattern | sort | uniq -c
Each tool:
Is stupid
Has no global objective
Optimizes locally
Can still produce dangerous global effects
That is instrumental convergence — but:
Without agency
Without values
Without intelligence
So the risk isn't "intelligence." It's compositional optimization under constraint.
Sandboxing precedes "AI containment"
chroot
jails
containers
SELinux
seccomp
All of these assume:
The process will try to escape
Intent doesn't matter
Constraints must be structural, not motivational
That directly contradicts large swaths of alignment rhetoric that say:
"If we instill the right values..."
Unix says:
"No. Assume betrayal. Design anyway."
That's not pessimism — it's engineering realism.
