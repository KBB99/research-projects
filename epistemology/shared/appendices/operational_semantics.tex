\section{Operational Semantics}

This appendix specifies execution dynamics without appealing to representation or intention.

\subsection{Execution Rule}

Let $x_t \in \States$ be the system state at time $t$.

\[
\ActionFn(x_t) =
\begin{cases}
a_t \in \Actions & \text{if } \ConstraintFn(x_t) = 1 \\
\Refusal & \text{if } \ConstraintFn(x_t) = 0
\end{cases}
\]

\subsection{State Evolution}

\begin{definition}[Dynamics]
State evolution is governed by
\[
x_{t+1} = \Dynamics(x_t, a_t)
\]
if $a_t \neq \Refusal$. If $a_t = \Refusal$, the system halts or defers.
\end{definition}

\subsection{Replay}

\begin{definition}[Replay]
Replay is the reconstruction map
\[
\ReplayFn : \Events \to \States
\]
such that identical event traces yield identical reconstructed states.
\end{definition}

Replay establishes semantic identity via history rather than configuration.

\subsection{Open vs Closed Loop}

An agent operates in closed loop: constraints are checked prior to execution.  
A didactic interface induces open-loop execution: constraints are evaluated only upstream.
