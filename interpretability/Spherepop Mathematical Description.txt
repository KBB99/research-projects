Spherepop (Event-Scope Calculus)
Spherepop is a history-first formal system in which
events, not states, are primitive
.
Let
𝔼 be a countable set of irreversible events
𝕊 be a set of nested scopes ("bubbles")
ℋ be the space of finite event histories
Each scope σ ∈ 𝕊 is defined by its history:
σ ≔ ⟨Hσ⟩, where Hσ ∈ ℋ
There is no global state. System evolution is a monotone sequence of histories:
H₀ ⊂ H₁ ⊂ H₂ ⊂ ···
An event e ∈ 𝔼 is a partial function on histories:
e : ℋ ⇀ ℋ
such that if e(H) is defined, then:
e(H) = H ∪ {e}
and e ∉ H
Thus all transitions are
irreversible and non-idempotent
.
Identity by History
Objects are not primitive; they are equivalence classes of histories.
For entities a, b:
a ≡ b  ⇔  Ha = Hb
Copying produces a new history, therefore a copy is never identical.
Scope as Geometry
Scopes form a rooted tree under inclusion:
(𝕊, ⊆)
Operations act on scopes as geometric transformations on history space.
Let σ ∈ 𝕊.
Pop:
Pop(σ) removes its boundary and merges histories:
Pop(σ) : Hparent ← Hparent ∪ Hσ
Bind:
Bind(σ₁, σ₂) creates a new scope with joint history:
Bind(σ₁, σ₂) = σ₃
Hσ₃ = Hσ₁ ∪ Hσ₂
All scope operations preserve history monotonicity.
Constructive Mereology
Part-whole relations are temporal, not set-theoretic.
x ⊑ y  ⇔  Hx ⊆ Hy
There is no unrestricted comprehension; parts exist only if constructed by events.
Computation
A computation is a path through history space:
C = (e₁, e₂, ..., eₙ)
evaluated as:
Hₙ = eₙ ∘ ··· ∘ e₁ (H₀)
Order is intrinsic. There is no global undo.
Summary Axiom
Spherepop replaces state with history:
What exists is what has happened.
Meaning, identity, and structure are functions of irreversible event geometry.
