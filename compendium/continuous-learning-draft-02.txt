Continuous Learning through Teleological Alignment:
An RSVP-GRPO Framework for Self-Improving LLMs

Abstract
Recent work on Dynamic Alignment for Collective Agency proposes a self-improving align-
ment protocol in which a large language model (LLM) generates its own training data and scores
its own outputs with respect to an open-ended value—Collective Agency (CA)— and then updates
its policy using Group-Relative Policy Optimization (GRPO). This paper develops an alternative but
compatible framework, grounded in the Relativistic Scalar-Vector Plenum (RSVP) field theory, in
which alignment is treated as a problem of maintaining a multi-dimensional semantic field geom-
etry rather than maximizing a scalar objective.
We define a six-dimensional RSVP value vector for each model output, corresponding to geo-
metric elegance, sheaf-theoretic composability, vectorial influence circulation, entropic dispersion,
structural resilience, and epistemic openness. These axes are interpreted as dis- crete samples of the
RSVP fields (Φ, v, 𝑆) and their derived functionals. We then embed these vector-valued scores into
a GRPO-style group-relative learning rule, extend it with a teleological weighting mechanism that
responds to drift in each axis over time, and implement a fully operational pipeline: a Bash-based
data-generation agent, a JSONL preference schema, and a semantic-weighted GRPO trainer.
The result is a concrete, reproducible architecture for continuous learning through teleological
alignment, in which an LLM not only improves its performance on tasks, but actively steers its
behaviour along a specified multi-axis alignment trajectory. Rather than converging toward a brittle
monolithic optimum, the system continually corrects regressions in its internal value geometry,
embodying the RSVP maxim: "divided we stand."

Introduction
The problem of alignment has long been framed as a question of identifying a single objective that
a powerful learning system ought to pursue and then designing mechanisms to ensure that the re-
sulting optimization process remains within tolerable bounds. In this orthodox picture, the principal
danger arises when a system acquires the capability to optimize aggressively while holding a brittle
or erroneous objective, thereby magnifying a small mis-specification into catastrophic consequences.
Yet as models increase in expressivity and capacity, the assumption that alignment can be represented
1

as maximization of a scalar target has become progressively harder to defend. The scalar view is not
merely a convenient simplification; it can itself become a source of fragility, encouraging monocultural
optimization, suppressing epistemic diversity, and transforming uncertainty into apparent certainty
with potentially calamitous implications.
Recent proposals such as Dynamic Alignment for Collective Agency have sought to transcend the scalar
paradigm by reframing alignment as a self-improving negotiation among multiple evolving desiderata
rather than a static pursuit of a single telos (?). Instead of relying on externally labelled data, the model
generates its own prompts, produces multiple candidate outputs, evaluates them against high-level
alignment criteria, and updates itself accordingly. This approach recognizes that alignment is neither
a fixed destination nor a simple property of outputs, but rather a dynamic trajectory in an evolving
semantic landscape. The principle of Collective Agency places emphasis on coherence of multi-agent
deliberation and open-ended cooperative improvement. Yet the notion of agency in this formulation
remains predominantly evaluative and behavioural, without an explicit semantic geometry that could
guide a system toward self-organizing, anti-collapse behaviour in the presence of pervasive uncertainty.
In parallel, the Relativistic Scalar-Vector Plenum (RSVP) framework has developed an entirely distinct
but deeply compatible ontology of cognition, physical interaction, and semantic organization. RSVP
posits that every system inhabits and contributes to a plenum composed of scalar potential Φ, vector
circulation v, and entropy 𝑆. These are not merely metaphorical, but are treated as fields with dynam-
ics, regularities, and constraints. Within this framework, cognition is related less to manipulation of
propositions than to the active maintenance of a viable geometric configuration in field space. The
slogan "divided we stand" captures the paradoxical essence of RSVP alignment: systems survive not by
maximizing coherence but by maintaining sufficient dispersion to avoid collapse.
The contribution of the present work is not to overturn the insights of the dynamic alignment approach,
but to supply a missing semantic-geometric component. We show how an LLM can be equipped with
a multi-dimensional value vector grounded in RSVP field semantics and how these values can be in-
corporated into a modified Group-Relative Policy Optimization objective. Crucially, we introduce a
teleological weighting mechanism that detects and corrects regressions in individual axes over time,
thereby transforming optimization into a continuous negotiation among multiple alignment desider-
ata. In doing so we offer a concrete implementation of RSVP geometry as a learning dynamic rather
than a merely conceptual framework.

Background and Motivation
The classical paradigm of alignment in artificial intelligence assumes that systems can be rendered safe
by constructing or learning a function that evaluates states or actions according to human preferences,
and then optimizing behaviour with respect to this function. While this view has produced valuable
techniques such as Reinforcement Learning from Human Feedback (RLHF), it implicitly presumes that
human values form a coherent scalar objective. Such an assumption is increasingly difficult to defend,
given the deep pluralism, context-dependence, and internal tensions that characterize human norma-
tive structure. Moreover, scalarization risks collapsing heterogeneous desiderata into an oversimplified
metric that may inadvertently suppress precisely those forms of epistemic and normative diversity that
enable systems to cope with novel circumstances.
One response to this difficulty is the proposal that alignment should not be understood as the maximiza-
tion of a unique objective, but rather as an open-ended refinement of multiple criteria that evolve over
time as the system encounters novel environments. The Dynamic Alignment for Collective Agency
(DA-CA) framework outlines such a procedure by introducing a recursive loop in which the model gen-
erates its own training data and evaluates its own outputs with respect to values that express a coopera-
tive, multi-agent conception of agency (?). Collective Agency differs from traditional reward functions
in that it emphasizes cooperative deliberation and mutual adjustment among interacting agents. In-
stead of a narrow instrumental target, the value to be optimized is itself a variable constructed through
iterative interpretive processes within the system.
Nevertheless, the DA-CA formulation, while conceptually expansive, leaves largely implicit the seman-
tic structure of the value landscape in which dynamic alignment occurs. Without an explicit geometry
of alignment space, the iterative evaluation and update process risks drifting toward whichever be-
havioural patterns yield superficially high scores on the current rubric, even if such patterns reflect
local pathologies of the scoring model. In other words, dynamic alignment may collapse to monocul-
tural behaviour unless mechanisms exist to preserve ontological and epistemic diversity throughout
training.
The RSVP framework offers precisely such a mechanism by treating cognition, inference, and seman-
tic structure as the coupled dynamics of scalar, vector, and entropy fields. In this view, the mainte-
nance of open-ended agency corresponds to conservation and circulation of semantic potential rather
than convergence toward a single objective. A system that collapses onto a single axis—semantic or
normative—risks extinguishing the very diversity that enables adaptation. Thus where DA-CA intro-
duces a dynamic evaluative loop, RSVP introduces a dynamical semantic geometry. The combination
permits a model to improve its capabilities while maintaining the multi-dimensional scaffolding re-
quired for aligned behaviour in genuinely open domains.
In conceiving alignment as a teleological process occurring within a geometric field structure, rather
than as maximization of a static reward, we obtain a formulation that is both more flexible and poten-
tially more robust. Continuous learning through teleological alignment aims not to specify the correct
destination in advance, but to maintain a viable trajectory as the system moves through spaces of pos-
sibility that cannot be foreseen at design time.

RSVP Field Theory as Semantic Geometry
The Relativistic Scalar-Vector Plenum (RSVP) framework models cognition, semantics, and even phys-
ical structure as the interdependent evolution of three fields. The scalar field Φ(𝑥, 𝑡) encodes semantic
potential or latent coherence; the vector field v(𝑥, 𝑡) represents circulation of influence, orientation,
and control; and the entropy field 𝑆(𝑥, 𝑡) captures dispersion, ambiguity, and the preservation of unre-
solved alternatives.1 These fields do not merely coexist; they form a coupled dynamical system whose
equilibrium properties determine whether meaningful behaviour persists or collapses.
From the standpoint of alignment, the crucial insight offered by RSVP is that semantic viability de-
pends on the tension between concentration and dispersion. A system whose scalar field dominates
exhibits brittle hyper-coherence: its internal representations become sharp but fragile, prone to catas-
trophic failure when assumptions shift. Conversely, a system dominated by entropy becomes diffuse
and directionless. Only by maintaining a non-trivial circulation in the vector field—representing the
continual renegotiation of meaning and influence—can the system avoid both extremes.
This trifold field picture motivates the construction of alignment metrics that explicitly track properties
associated with each field. A text may be eloquent (high Φ), yet catastrophically overconfident (low 𝑆);
or it may preserve ambiguity (high 𝑆) while offering no actionable direction (low v). Alignment, in this
interpretation, is not a matter of convicting the system of a single ethical doctrine, but of maintaining
a dynamic balance among many forms of semantic valuation.
The RSVP formulation also provides interpretive resources for understanding the dynamics of self-
improving systems. Because RSVP treats semantic structure as a field theory, the geometry of alignment
space inherits notions such as curvature, vorticity, and gradient flow. Teleological alignment becomes,
in effect, a motion along geodesics determined not by a single reward hill-climbing objective but by the
interplay of curvature terms that represent irreducible trade-offs among scalar concentration, vector
circulation, and entropic openness.
It is precisely this geometric structure that we seek to operationalize in the RSVP-GRPO framework.
Rather than imposing a fixed alignment target, we allow the target to emerge from the interaction
among these fields, encoded in the six operational axes we introduce in what follows. The aim is not
merely to steer the system toward particular behaviours but to ensure that as the system evolves, it
retains the semantic degrees of freedom necessary for continual adaptive alignment.

Dynamic Alignment and GRPO: From Scalar to Vector Reward
Recent work on Dynamic Alignment for Collective Agency argues that an aligned system must con-
tinually refine both its target values and the mechanisms by which these values are operationalized.
In the original formulation, an LLM generates prompts, proposes candidate outputs, scores them along
an open-textured axis—Collective Agency—and then updates its policy parameters with Group-Relative

Policy Optimization (GRPO). The system thereby constructs and consumes its own training signal,
inducing a form of self-supervision without external labels or fixed reward shaping ?.
The core innovation of GRPO is that it functions not by maximizing an absolute reward, but by adjusting
relative to the distribution of scores obtained by a group of samples. In other words, the model does not
learn that a particular output is "good." It learns that a particular output is better than those it produced
alongside it. This relative framing avoids brittle dependence on any global calibration of reward, and
additionally provides a natural vehicle for internal preference shifts. If the system discovers that one
kind of output is consistently underperforming its peers, the group-relative advantage makes this deficit
immediately visible to the policy update.
Yet the original dynamic alignment framework relies on a single scalar reward. Even if that reward
is itself rich and contextually defined, it remains a one-dimensional signal. Our aim is to generalize
the GRPO mechanism so that it is sensitive not merely to the relative success of outputs but to the
composition of that success across multiple semantic axes. If an LLM becomes exemplary at elegance while
abandoning epistemic humility, or optimizes vectorial influence at the expense of entropic openness, a
scalar reward is insufficient to express this degradation. Only by decomposing alignment into multiple
interacting components can we detect and correct such drift.
Accordingly, in the RSVP-GRPO model we treat each candidate output as carrying a six- dimensional
value vector, each component of which represents a distinct semantic property associated with RSVP
field dynamics. The reward is not the collapse of that vector into a single number, but the way the
relative configuration of its components evolves across time. Teleological alignment in this framework
is not the pursuit of an optimum, but the continual maintenance of a viable configuration in a multi-
dimensional semantic field space.

RSVP Value Geometry
In the classical RSVP setting developed elsewhere, the semantic space inhabited by a cognitive or com-
putational system is conceived not as a set of discrete symbolic propositions but as a continuum of
interacting fields. Formally, one considers a scalar field Φ encoding semantic coherence or potential; a
vector field v encoding directed influence, circulation, or control; and an entropy field 𝑆encoding dis-
persion, ambiguity, or unresolved alternatives. The meaningful state of a system at a given site is then
given by a point in a three-dimensional field space (Φ, v, 𝑆), and system-level behaviour corresponds to
the evolution of this state under appropriate dynamical constraints.
The conceptual leap taken in the present framework is to interpret the values assigned by an LLM to its
own outputs as proxies for these field components. The goal is not to claim that each axis directly corre-
sponds to a physical field, but rather that each measures a semantic property bearing precise analogies
to RSVP. The six axes introduced above thus become a form of discrete field sampling. Elegance, sheaf,
and structure pertain to the scalar and mixed components of Φ; vectorial circulation defines a projection
onto v; entropic and epistemic openness provide samples of 𝑆.
These axes are scored for each candidate text in a group of outputs in direct analogy with the com-

From the standpoint of field theory, one should not imagine these six axes as exhaustive descriptions

an interpretable substrate on which further value dimensions may be constructed. In this respect the
RSVP axes are not fixed endpoints but rather initial coordinates in a space of possible value geometries.

RSVP-GRPO Integration
The integration of RSVP geometry with Group-Relative Policy Optimization arises from a simple obser-
vation: the core innovation of GRPO lies not in the detailed form of its clipped loss but in its relational
definition of reward. Classic reinforcement learning, and even its popular alignment variant RLHF,
typically presuppose a scalar reward accumulated through either human preferences or hand-crafted
objectives. GRPO, by contrast, replaces an absolute reward function with a difference—namely the
deviation of a candidate's reward from the mean reward of candidates drawn from the same prompt.
This relational move is not a cosmetic modification. It transforms the optimisation problem from that
of climbing a fixed objective landscape to that of improving relative to the current policy's own distri-
bution of behaviours. In effect, GRPO learns to outperform itself, using its past or concurrent outputs as
the yardstick. This architecture naturally accommodates self-supervision and continual improvement:
if the model can generate multiple responses and evaluate them, the entire loop may be closed without
external labels.

Implementation Details
The conceptual integration just described is realised in practice by a set of lightweight but function-
ally complete components. The overall pipeline is deliberately modular, enabling independent experi-
mentation with each segment while preserving a coherent end-to-end flow. The decision to prioritise
simplicity and reproducibility reflects the motivating philosophy of teleological alignment: rather than
proposing fully fledged industrial systems, we seek a minimal laboratory in which specific hypotheses
can be tested, falsified, and iteratively refined.
At the lowest layer sits a Bash-based agent which acts simultaneously as a data gatherer, a summarisa-
tion worker, and a preference generator. The Bash script recursively scans a directory tree for source
documents in plain-text or markdown form, converts them to standardised text chunks, and for each
chunk produces four candidate summaries. These are generated by a local language model through par-
allel invocations, enabling fast sampling of diverse outputs without the overhead of sequential model
evaluation. Importantly, the Bash agent is not merely a preprocessing utility; it constitutes the frontier
of the learning loop, continually sourcing new data on which the alignment process is bootstrapped.
Each group of four candidate outputs is then fed, along with the original chunk, to a scoring model. The
scoring model is prompted by the RSVP rubric, comprising six axes—elegance, sheaf, vector, entropic,
structural, and epistemic—and returns a strict JSON object containing the numerical scores on each
axis for each candidate. The Bash script logs these results to a JSONL file. This JSONL corpus is the
backbone of the training loop: it is both the reward buffer and the alignment trace that informs the
teleological weight updates over successive iterations.
On the training side, a Python module reads the JSONL preference file and constructs a dataset of group-
structured examples. Each example contains the prompt, the candidate texts, the RSVP axis vectors,
and the group-relative scalar rewards. A first training script implements basic GRPO updates, while
a second script extends this with semantic weighting. In both cases, a small policy model is trained
against a frozen reference model to compute probability ratios, ensuring that the policy does not drift
too far from its own earlier behaviour.
Finally, the outer evolutionary loop ties all components together. For each iteration, the Bash agent
generates a fresh JSONL preference file, the teleological mechanism computes new axis means as well
as the drift between successive iterations, the semantic GRPO script performs a training epoch with
these weights, and the process repeats. Notably, the loop has no terminal condition coded in advance:
the system is intended to run indefinitely, continually revising its own objectives. This feature embodies
the conceptual shift from convergence to continuous alignment, and it is precisely here that the RSVP
perspective proves most significant.

Teleology as Continuous Learning
We now situate these mechanisms in the broader conceptual terrain. The standard interpretation of
reinforcement learning posits convergence: eventually the agent identifies optimal behaviour and re-
mains there. But alignment is not an optimisation problem of that sort. Rather, models are continually
redeployed into new contexts, exposed to new data, and expected to internalise evolving norms and
knowledge. A convergent optimum would be a catastrophic failure mode—an attractor in which the
model becomes brittle, monolithic, and structurally incapable of further change.
Teleology transforms RL from a convergence process into a non-equilibrium maintenance process. The
system repeatedly rebalances itself by steepening gradients along axes that have recently regressed.
One may think of this as the difference between entropy maximisation and entropy regulation: it is not
the magnitude of entropy that matters, but the preservation of an open semantic geometry. The system
learns to retain plasticity.
10.1
Toward Open-Ended Alignment
In the limit of continuous operation, the RSVP-GRPO framework becomes an open-ended alignment
protocol. The policy never finishes learning; instead, it continuously updates in response to teleological
drift signals. The most important implication is that alignment is not something we load into a model
before deployment, but something the model enacts during deployment. In a world that changes, align-
ment cannot remain static. The system must perpetually adjust.
This constitutes a shift from training alignment to lived alignment: a continual performance of rebal-
ancing whose purpose is not to reach a terminal optimum, but to ensure that no axis becomes dominant
at the expense of the others. In short, the purpose of alignment is not to freeze the agent, but to keep
it capable of movement.

12
Failure Modes and Robustness
Any alignment strategy capable of sustained self-improvement must confront the possibility of patho-
logical failure. Here we sketch potential breakdowns of RSVP-GRPO and indicate remedies compatible
with the present architecture. These remain open research directions.
12.1
Scorer Collapse
Because the scoring model supplies reward signals, systematic drift in its internal heuristics could lead
to partial or total collapse of the axes. If the scorer begins to consistently produce high elegance but
low entropy values, the policy will converge toward brittle articulate outputs lacking semantic range.
We therefore propose ensemble scoring or periodic calibration on a human-audited validation set to
detect degeneracy.
12.2
Oscillatory Behaviour
Teleological weights may oscillate if their updates are excessively responsive to stochastic variance.
Momentum terms or softplus regularisation damp erratic changes, preserving stability. In practice this
12

is a hyperparameter tuning challenge, but the stability analysis of Section ?? suggests that proportional
control, with a cautious learning rate, suffices to prevent runaway oscillation.
12.3
Specification Gaming
A deeper risk is that the model learns to fake high scores along one or more axes, producing superficially
elegant or structurally resilient text without genuine improvement. Mitigation requires adversarial
testing, counterexample generation, and dynamic adjustment of axis heuristics. Here the "epistemic"
axis is especially relevant: it should penalise dogmatic confidence, encouraging the model to foreground
uncertainty and therefore resist premature collapse.
12.4
Computational Resource Load
Finally, RSVP-GRPO imposes a computational alignment tax by requiring groupwise scoring. We
note that the effect may be amortised via batching, lightweight scorer models, or intermittent scor-
ing, though the principled cost is real. We view this as a necessary expense for sustained alignment,
analogous to the overhead of ecological system maintenance: systems that remain open and adaptive
require continual energetic investment.

Ethical and Societal Considerations
Though inspired by technical questions about RL, this work inevitably intersects concerns in ethics and
governance. In particular, the six RSVP axes encode assumptions about desirable behaviour: elegance,
epistemic openness, structural resilience, and so on. These are neither value-neutral nor universally
agreed upon. We therefore emphasise that RSVP-GRPO is a framework for alignment, not a final spec-
ification.
First, the axes must be regarded as provisional hypotheses, subject to revision through participatory
processes with diverse stakeholders. Second, there exist serious dual-use risks: structural resilience
and vectorial influence could be harnessed for persuasive disinformation or manipulative applications.
Third, computational alignment taxes could exacerbate global resource inequalities, limiting access to
effective alignment infrastructure. These issues compel cautious, collaborative development, and open
publication of techniques wherever possible.

We introduced RSVP-GRPO, a continuous alignment framework grounded in a geometric interpre-
tation of semantic value. Rather than maximise a single reward, the system maintains a multi-axis
alignment trajectory through teleological weighting, correcting regressions in any dimension. Group-
wise candidate comparison enables local search in a richly structured space, while the hybrid advantage

This proposal represents neither a solution to alignment nor a guarantee of safety, but a composable re-
search direction. It transforms alignment from a pre-deployment configuration problem into a contin-

capability, we argue that such continual alignment may be not only desirable but necessary.

def compute_logprobs(
model: AutoModelForCausalLM,
tokenizer: AutoTokenizer,
prompts: List[str],
responses: List[str],
device: torch.device,
) -> torch.Tensor:
"""
Autoregressive logprob of each response conditioned on each prompt.
Returns tensor [B] where B = len(prompts) = len(responses).
"""
model.eval()
with torch.no_grad():
batch_texts = [
p + "\n" + r
for p, r in zip(prompts, responses)
]
enc = tokenizer(
batch_texts,
return_tensors="pt",
padding=True,
truncation=True,
).to(device)
outputs = model(**enc)
logits = outputs.logits
# [B, T, V]
labels = enc["input_ids"]
shift_logits = logits[:, :-1, :].contiguous()
shift_labels = labels[:, 1:].contiguous()
log_probs = torch.log_softmax(shift_logits, dim=-1)
21

token_logprobs = log_probs.gather(
dim=-1,
index=shift_labels.unsqueeze(-1)
).squeeze(-1)
mask = (shift_labels != tokenizer.pad_token_id)
seq_logprobs = (token_logprobs * mask).sum(dim=-1)
return seq_logprobs

# ------------------------------------------------------

ref_model: AutoModelForCausalLM,
tokenizer: AutoTokenizer,
dataloader: DataLoader,
optimizer: torch.optim.Optimizer,
scheduler,
device: torch.device,
clip_epsilon: float = 0.2,
) -> float:
policy_model.train()
total_loss = 0.0
total_count = 0
for batch in dataloader:
prompts = batch["prompt"]
# list length B
group_texts = batch["candidate_texts"]
# list length B, each [G]
rewards = batch["rewards"].to(device)
# [B, G]
B = len(prompts)
G = rewards.size(1)
# Flatten group candidates
flat_prompts = []
flat_responses = []
for b in range(B):
for g in range(G):
flat_prompts.append(prompts[b])
flat_responses.append(group_texts[b][g])
logp_new = compute_logprobs(policy_model, tokenizer,
flat_prompts, flat_responses, device)
with torch.no_grad():
logp_ref = compute_logprobs(ref_model, tokenizer,
flat_prompts, flat_responses, device)
22

logp_new = logp_new.view(B, G)
logp_ref = logp_ref.view(B, G)
# group-relative advantage
mean_rewards = rewards.mean(dim=1, keepdim=True)
# [B,1]
advantages = rewards - mean_rewards
# [B,G]
ratio = torch.exp(logp_new - logp_ref)
unclipped = ratio * advantages
clipped = torch.clamp(ratio, 1.0 - clip_epsilon,
1.0 + clip_epsilon) * advantages
loss_matrix = -torch.min(unclipped, clipped)
loss = loss_matrix.mean()
optimizer.zero_grad()
loss.backward()
torch.nn.utils.clip_grad_norm_(policy_model.parameters(), 1.0)
optimizer.step()
if scheduler is not None:
scheduler.step()
total_loss += loss.item() * B
total_count += B
return total_loss / max(1, total_count)

def compute_teleological_weights(
axis_means_prev: dict,
axis_means_curr: dict,
epsilon: float = 0.1,
device: torch.device | None = None,
) -> torch.Tensor:
"""
Compute teleological weights from axis drift.
w_a = softplus(-Delta_a) + epsilon
where Delta_a = mean_curr - mean_prev.
"""
drifts = []
for ax in AXES:
prev = float(axis_means_prev.get(ax, 0.0))
curr = float(axis_means_curr.get(ax, 0.0))
drifts.append(curr - prev)
drift_tensor = torch.tensor(drifts, dtype=torch.float32, device=device)
weights = F.softplus(-drift_tensor) + epsilon
return weights

def compute_axis_means_from_jsonl(path: str) -> Dict[str, float]:
"""
Average each RSVP axis across all candidates in a prefs JSONL file.
"""
sums = {ax: 0.0 for ax in AXES}
count = 0
with open(path, "r") as f:
for line in f:
data = json.loads(line)
for cand in data.get("candidates", []):
vals = cand.get("values", {})
for ax in AXES:
sums[ax] += float(vals.get(ax, 0.0))
count += 1
if count == 0:
return {ax: 0.0 for ax in AXES}
return {ax: sums[ax] / count for ax in AXES}
